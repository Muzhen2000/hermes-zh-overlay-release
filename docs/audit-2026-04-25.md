## 成功路径

1. 先锁定官方 commit，再从干净官方源码生成 patch。
2. 先用外置 YAML 数据覆盖，数据覆盖不到时才加最薄 `_ui()` 取词钩子。
3. 按类别扫描，不按单张截图修补。
4. 终端和 Discord 要分开验证；Discord 还要验证服务端 slash command 说明是否真正同步。
5. 文档、manifest、skill、README 必须和真实 patch 面一致。
6. 远端仓库只保留当前可验证 release，旧补丁保留在 git 历史中，不保留为可误用目录。

## 失败经验

- 只修 terminal 或只修显式 `@tree.command` 会漏掉 Discord 动态命令说明。
- “零 patch”文档如果没有及时更新，会让后续 agent 误判真实源码面。
- Telegram / 飞书历史 release 留在仓库，会诱导新线程把旧范围当成当前范围。
- 在已汉化脏树上继续补丁会扩大不可解释 diff；必须重新回到官方基线导出。
- Discord 客户端 UI 的 slash command cache 和服务端 command sync 不是一回事，必须用 server-side payload 或实际 UI 验证。
